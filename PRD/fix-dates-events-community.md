## Progress Update as of [June 30, 2026 — 6:55 PM Pacific]

### Summary of changes since last update
First entry for this branch. Fixed the timezone/date + events/community cluster from the automated audit: 9 verified findings plus 2 live-test issues (calendar opening on the wrong month/"today"). The unifying theme — date-only values and all-day events must render back as the same calendar day the user meant, for viewers west of UTC (all US zones). Established a coherent calendar date model: all-day events store at UTC midnight and render with UTC getters; "today"/current-month derive from the viewer's LOCAL clock; timed events stay local.

### Detail of changes made
- **[#1] Community "valid until" one day early + early expiry** — `lib/ask-validate.ts` `validateValidUntil` now parses a date-only `YYYY-MM-DD` as LOCAL end-of-day (`T23:59:59.999`) instead of relying on Date.parse's UTC-midnight rule. Full ISO timestamps still pass through unchanged. This fixes both the "Jun 29 for a June 30 entry" display (board/detail render with local `toLocaleDateString`) and the `isExpired` early-expiry (`lib/exchange.ts`, unchanged — it just reads the now-correct instant).
- **[#2 + live-test] All-day/OHS calendar events one day early** — `lib/events/calendar.ts`: added `utcDayKey` (UTC getters) and made `eventOverlapsDay` compare all-day events by UTC calendar day (inclusive start→end range via lexicographic YYYY-MM-DD keys). Timed events keep the local instant overlap. `app/(authed)/events/event-bits.tsx` `formatEventWhen` formats all-day events with UTC getters + `timeZone:'UTC'`. `events-calendar-client.tsx` `DateBlock` uses `getUTCDate()` + `timeZone:'UTC'` for all-day. `.ics` export was already UTC-correct (untouched).
- **[live-test] Calendar opens on wrong month / wrong "today"** — root cause: `EventsCalendarClient` is server-prerendered (UTC on Vercel) and the `now`/`anchor` state initializers baked in the UTC date, which doesn't self-correct on hydration. Now seeds from `new Date()` and re-syncs `now` + `anchor` to the client's true LOCAL clock in a mount `useEffect`. `isToday`/anchor now reflect the viewer's local date consistently with the all-day fix.
- **[#3] Pill click opened whole-day drawer** — `events-calendar-client.tsx`: the day cell was a `<button>` with nested `<button>` pills (invalid HTML) and the pill click bubbled to the day handler, which cleared the selected event. Restructured: day cell is now a `<div>` with a full-cell "pick day" `<button>` laid BEHIND the content (z-0) and the number + pills as siblings on top (z-10). `DayEventPill` also calls `e.stopPropagation()`. A pill click now opens that event's DetailDrawer.
- **[#4 + #7] Proposed meeting times rendered in UTC** — `lib/community-schedule.ts` `formatSlot` now takes an explicit `timeZone` (default `OHS_SCHEDULE_TZ = "America/Los_Angeles"`) passed to `toLocaleString`, keeping the "short" tz label (e.g. "3:00 PM PDT"). Fixes both the intro email (`community/actions.ts:593`) and the community post page (`community/[id]/page.tsx:608`) — both call `formatSlot` with no override and inherit the fix.
- **[#5] "My posts" filter showed everyone's posts when viewer has zero posts** — `lib/exchange.ts`: dropped the `.size > 0` guard on the mine-only branch so an empty id set correctly yields zero posts + the empty state.
- **[#6] Editing an all-day event shifted its date off UTC servers** — `app/(authed)/events/[id]/edit/page.tsx`: renamed `localParts` → `eventParts(iso, allDay)`; all-day events now derive the form date from UTC getters (matching storage), timed events keep local getters. Deterministic regardless of server zone.
- **[#8] Joiners list emitted `<a>` directly in `<ul>`** — `community/[id]/page.tsx`: wrapped the linked chip in an `<li>` (matching the no-link branch) so list semantics are valid for assistive tech.
- **[#9] All-day event with end == start rejected** — `lib/events/validate.ts` `validateRange(start, end, allDay=false)`: for all-day events the end is the inclusive last day, so end === start is allowed (end >= start) with day-oriented copy ("The end date must be on or after the start date"). Timed events still require end strictly after start. Caller `events/actions.ts` passes `allDay`.
